# pi-git — Build Spec

> Design/implementation spec. Decisions, responsibilities, interfaces, build
> order, acceptance criteria. No function bodies.

**Goal:** Non-destructive checkpointing with rollback, git worktrees for isolated parallel work, optional auto-commit, and a guard against destructive git/fs commands.

**Architecture:** A thin `git` exec wrapper (all git plumbing, injectable for tests) under three feature modules — checkpoints, worktrees, and a pure command-guard classifier — wired through tools, lifecycle hooks, and commands.

**Tech Stack:** TypeScript/ESM, Bun + `bun test`, `node:child_process` (or `pi.exec`) for git, `@earendil-works/pi-*` peers.

## Global Constraints

See `pi-shared/HOUSE-STYLE.md`. Application:
- Tools `git_checkpoint`, `git_rollback`, `git_worktree_new`, `git_worktree_list`, `git_worktree_remove`.
- Emits `git:checkpoint { ref, reason }`, `git:rollback { ref }`. Subscribes `todo:task-complete`.
- Enforcement dial applies (auto-checkpoint + destructive-guard). Commands `/pi-git` (config) and `/rollback` (human override). Log `[pi-git]`.

## Design Decisions

1. **Checkpoints as commit objects on a private ref namespace**, not stashes. Snapshot everything (tracked + untracked) without touching the working tree or index: stage into a temporary index (`GIT_INDEX_FILE`), `git add -A`, `git write-tree`, `git commit-tree` → commit sha, store under `refs/pi-git/checkpoints/<id>`. This is robust where `git stash create` (tracked-only) is not. Rollback = restore that tree into the working tree.
2. **Auto-checkpoint on `turn_start`** (in `notify`/`block` mode): create a checkpoint labeled with the turn, record `{ id, ref, reason, createdAtIso }` via `appendEntry`, emit `git:checkpoint`. Also checkpoint on `todo:task-complete` (reason `"task complete"`).
3. **Rollback** — `git_rollback({ checkpoint_id? })` tool (agent undoes its own mess; defaults to latest) and `/rollback [id]` command (human). Restores the checkpoint tree; before overwriting, snapshots current state as a safety checkpoint.
4. **Worktrees** wrap `git worktree add/list/remove` — the isolation primitive pi-spawn composes with for parallel agents. Worktrees live under a configured base dir.
5. **Destructive guard** — `pi.on("tool_call")` classifies `bash` commands; `git reset --hard`, `git clean -f[d]`, `git push --force/-f`, `git checkout -- .`, `rm -rf` etc. In `notify` → `ctx.ui.confirm`; in `block` → `{ block: true, reason }`; in `off` → allow. The classifier is pure and the highest-value test target.
6. **Auto-commit (optional, off by default)** — on `agent_settled`, if enabled, commit working changes with a message derived from the last assistant message.

### Open items for the human
- Default `mode` (recommend `notify`). Worktree base dir. Whether auto-commit ships in v1 (recommend: off, tool-only).

## Interfaces / Contracts

```typescript
// git.ts — every method takes an injected exec; returns typed results
interface Worktree { path: string; branch: string; head: string; }
interface Git {
  snapshotTree(): Promise<string>;            // commit sha capturing tracked+untracked
  restoreTree(sha: string): Promise<void>;    // overwrite working tree from sha
  commitAll(message: string): Promise<string>;
  isDirty(): Promise<boolean>;
  currentBranch(): Promise<string>;
  updateRef(ref: string, sha: string): Promise<void>;
  listRefs(prefix: string): Promise<Array<{ ref: string; sha: string }>>;
  worktreeAdd(branch: string, path: string): Promise<string>;
  worktreeList(): Promise<Worktree[]>;
  worktreeRemove(path: string): Promise<void>;
}
function createGit(exec: ExecFn, cwd: string): Git;

// checkpoints.ts
interface Checkpoint { id: string; ref: string; reason: string; createdAtIso: string; }
function createCheckpoint(git: Git, reason: string, now: string): Promise<Checkpoint>;
function listCheckpoints(ctx: unknown): Checkpoint[];   // from appendEntry history
function restoreCheckpoint(git: Git, cp: Checkpoint): Promise<void>;

// guard.ts (pure)
interface Verdict { dangerous: boolean; rule?: string; }
function classifyCommand(command: string): Verdict;

// tools: git_checkpoint({reason?}), git_rollback({checkpoint_id?}),
//        git_worktree_new({branch, path?}), git_worktree_list(), git_worktree_remove({path})
// events: git:checkpoint {ref, reason}, git:rollback {ref}
```

## Build Sequence

- [ ] **1 · Scaffold** — per house-style. Accept: `bun test` runs.
- [ ] **2 · `classifyCommand` guard** — Accept: flags `git reset --hard`, `git clean -fd`, `git push --force`, `git checkout -- .`, `rm -rf /path`, with the matching `rule`; does NOT flag `git status`, `git reset` (soft), `git commit`, `rm file.txt`. (Largest test set; security-relevant.)
- [ ] **3 · `git.ts` snapshot/restore** — Accept (integration test in a temp repo): `snapshotTree` then dirty the tree then `restoreTree` returns the exact prior content, including a previously-untracked file.
- [ ] **4 · Checkpoints** — Accept: `createCheckpoint` writes a ref + persists metadata; `listCheckpoints` reads them back ordered; `restoreCheckpoint` round-trips (temp repo).
- [ ] **5 · Worktrees** — Accept (temp repo): `worktreeAdd` creates a linked worktree on a new branch; `worktreeList` includes it; `worktreeRemove` removes it.
- [ ] **6 · Tools** — Accept: each tool calls the right core fn and emits the right event; `git_rollback` with no id restores the latest checkpoint.
- [ ] **7 · Hooks + config** — Accept: `turn_start` auto-checkpoints in `notify`/`block`; `tool_call` guard confirms/blocks per `mode`; `todo:task-complete` triggers a checkpoint; `/pi-git` sets `mode`, `/rollback` restores.

## Test Strategy

`classifyCommand` — exhaustive pure unit tests. `git.ts`/checkpoints/worktrees — integration tests against a temp git repo created in a tmp dir (real git, isolated). Tools/hooks — fake `Git` + captured events. Auto-commit message derivation — pure.

## Risks / Notes

- Temporary-index snapshotting (`GIT_INDEX_FILE`) must not disturb the user's real index — verify in the step-3 integration test.
- Large untracked artifacts inflate checkpoints; consider honoring `.gitignore` in `snapshotTree` (decision to confirm at build time).
- Rollback is destructive to uncommitted work — always take the pre-rollback safety checkpoint (Decision 3).
